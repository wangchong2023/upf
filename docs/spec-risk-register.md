## 2. 风险管理全生命周期 (Risk Management Lifecycle)
风险项的处理流程遵循 IPD 闭环逻辑，与需求处理类似但侧重于“减缓与消除”：

1. **识别 (Identification)**:
   - 来源：TR 评审、QA 审计、测试失败、Agent 风险发现。
   - 动作：PM/SE 记录至矩阵，分配 `Risk ID`。
2. **评估 (Assessment)**:
   - 维度：影响范围 (Scope) x 发生概率 (Probability)。
   - 动作：确定风险级别 (Critical/High/Medium/Low)。
3. **应对 (Treatment)**:
   - 策略：规避 (Avoidance)、减轻 (Mitigation)、转移 (Transfer) 或接受 (Acceptance)。
   - 动作：SE/Architect 制定应对计划，分配负责人。
4. **追踪 (Tracking)**:
   - 频率：每周一由 PM 刷新。
   - 动作：同步状态 (Open -> In Progress -> Fixed -> Verified)。
5. **闭环 (Closure)**:
   - 准则：QA 专家进行回归审计，验证风险应对策略有效。
   - 动作：状态转为 **Closed**。

## 3. 治理准则
- **评估周期**: 每次里程碑切换 (`stage-next`) 前，必须由 PM/QA 联合审查风险矩阵。
- **降级机制**: 若风险状态为 "Critical" 且无有效应对策略，TR 评审应直接判定为 "No-Go"。
- **闭环审计**: QA 专家负责对风险应对策略的执行结果进行闭环审计。
