## 3. C 语言安全红线 (Security)
- **初始化**: 变量声明必须赋初值。
- **函数禁止项**: 严禁使用 `strcpy`, `strcat`, `sprintf`, `gets`。

## 4. JS/Node.js 深度规约
- **严格相等**: 强制使用 `===` 和 `!==`。
- **防御性编程**: I/O 操作必须包裹在 `try-catch` 块中。
- **配置中心化**: 严禁硬编码路径或指标，必须引用 `mgr-config.js`。

## 5. 构建系统规约 (Makefile)
- **工具抽象**: 必须使用变量 (如 `$(NODE)`)。
- **伪目标**: 逻辑目标必须声明为 `.PHONY`。
