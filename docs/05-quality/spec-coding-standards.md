# 5G UPF 代码编写规约速查表 (Cheat Sheet v3.1)

## 1. 基础命名 (Naming Convention)
| 对象 | 规范 | 示例 |
| :--- | :--- | :--- |
| **C 变量/函数** | `snake_case` | `pdu_session`, `get_buffer()` |
| **JS 变量/函数** | `camelCase` | `milestoneStatus`, `runAudit()` |
| **常量** | `UPPER_SNAKE_CASE` | `MAX_GTP_PAYLOAD`, `MIN_COVERAGE` |

## 2. 华为 C 语言排版准则 (Layout)
- **大括号换行**: 左大括号 `{` 必须单独起一行。
- **括号完备性**: 逻辑表达式必须使用完备括号：`if ((a == b) && (c == d))`。

## 3. 开源合规基线 (OSS Compliance)
- **禁止协议 (Tier 1)**: 严禁引入 GPL, AGPL, SSPL 等强传染性协议。违者 TR6 审计一票否决。
- **约束协议 (Tier 2)**: LGPL, MPL 等需经过 ARCHITECT 评审物理链接方式（必须动态链接/进程隔离）。
- **义务履行 (Tier 3)**: MIT, Apache, BSD 等友好协议必须在 `docs/05-quality/OSS_ATTRIBUTIONS.md` 登记以履行致谢义务。

## 4. JS/Node.js 深度规约
- **现代语法**: 严禁使用 `var`，统一使用 `const` 或 `let`。
- **配置中心化**: 严禁硬编码路径或指标，必须引用 `mgr-config.js`。

## 5. 构建系统规约 (Makefile)
- **工具抽象**: 必须定义为变量 (如 `$(NODE)`)。
- **缩进规范**: 指令块必须使用 `Tab` 字符缩进。
