## 2. CBB 资产库治理 (CBB Governance)
为提升研发 ROI，所有通用组件必须平台化沉淀：
- **物理隔离**: CBB 必须存放在 `src/lib-cbb/` 下，严禁与业务逻辑强耦合。
- **独立验证**: 每个 CBB 模块必须具备 100% 的单元测试覆盖率。
- **优先复用**: 架构设计 (ADR) 评审第一优先级是核实是否有可用的 CBB。
- **贡献激励**: 鼓励将成熟的业务组件重构成 CBB 并贡献至资产库。

## 3. 质量度量红线
- **CBB 复用率**: `(CBB 代码行数 / 总代码行数)` 必须 > 10%。
- **用例密度**: 20 个/KLOC。
- **代码覆盖率**: > 80%。

## 4. JS/Node.js 深度规约
- **配置中心化**: 严禁硬编码路径，必须引用 `mgr-config.js`。
