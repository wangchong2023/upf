# 5G UPF 代码编写规约速查表 (Cheat Sheet v1.0)

## 1. 基础命名 (Naming Convention)
| 对象 | 规范 | 示例 |
| :--- | :--- | :--- |
| **变量/函数** | `snake_case` | `pdu_session`, `get_buffer()` |
| **结构体/类型** | `snake_case` + `_t` | `session_context_t` |
| **宏/常量** | 全大写 + 下划线 | `MAX_GTP_PAYLOAD` |
| **全局变量** | `g_` + `snake_case` | `g_total_packets` |

## 2. 华为排版准则 (Layout)
- **大括号换行**: 左大括号 `{` 必须单独起一行。
  ```c
  if ((NULL == ptr))
  {
      do_action();
  }
  ```
- **括号完备性**: 逻辑表达式必须使用完备括号：`if ((a == b) && (c == d))`。
- **恒等风格**: 优先使用左常量比较：`if (NULL == p_buf)`。

## 3. C 语言安全红线 (Security)
- **初始化**: 变量声明必须赋初值：`uint32_t count = 0;`。
- **函数禁止项**: 严禁使用 `strcpy`, `strcat`, `sprintf`, `gets`。
- **函数推荐项**: 必须使用 `strncpy_s`, `strncat_s`, `snprintf_s`, `fgets`。
- **内存清零**: 敏感数据释放前强制 `memset_s`。

## 4. 高性能 VPP 规约 (Performance)
- **Dual-Loop**: 必须实现四报文循环处理以优化 SIMD。
- **Prefetch**: 处理当前报文时，必须预取后续报文 Header。
- **Zero-Copy**: 转发面严禁不必要的 `memcpy`。

## 5. 质量准出 (Quality Gate)
- **TDD**: 先写失败的测试，再写实现代码。
- **ASan**: `make quality-gate` 必须 0 内存错误。
- **Linter**: `cppcheck` & `clang-tidy` 必须 0 警告。
