# ADR: GTP-U 扩展报头解析架构决策 [SR.UPF.009.01.001]

## 1. 背景
5G 核心网在 N3 接口引入了 PDU Session Container 扩展报头，用于透传 QFI 等关键元数据。

## 2. 方案权衡
*   **Option 1: 逐包线性解析**: 实现简单，但在多扩展报头场景下 CPU 抖动大。
*   **Option 2: 向量化并行偏移 (Selected)**: 利用 VPP 向量化优势，预计算偏移量。
    - **优点**: 处理性能恒定 (O(1))，时延抖动 < 50ns。
    - **代价**: 增加了结构体内存对齐的复杂度。

## 3. 架构决策
采用 **Option 2**。解析逻辑物理实现在 `upf_gtpu_decap.c` 的内部函数中，结果通过 `vnet_buffer` 扩展位传递。

## 4. 后果
- **开发**: 需要严格遵循 DPDK 内存对齐规范。
- **测试**: 必须包含 100Gbps 压力下的稳定性验证。
