# 架构视图设计指南 (view-4p1)

## 架构层级、视图与需求映射矩阵 (体现方法)

| 架构层级 | 4+1 对应视图 | 承载的需求级别 | 体现重点 |
| :--- | :--- | :--- | :--- |
| **L0 (系统上下文)** | 场景视图 (+1) | RR (原始需求) | 业务边界、外部接口协议映射 (如 N2/N3/N4)。 |
| **L1 (容器/微服务)** | 逻辑视图 / 物理视图 | IR (特性需求) | 服务边界 (FUN)、部署位置、弹性伸缩策略 (PERF)。 |
| **L2 (组件/模块)** | 开发视图 / 进程视图 | SR/AR (规格/分配需求) | 核心类/函数设计、并发模型、同步/异步调用流。 |

## 各层级视图设计准则

### L0: 场景视图 (+1)
- **重点**：描述 UPF 作为一个整体在 5G 网络中的位置。
- **关联需求**：RR。
- **输出**：外部接口协议栈 (3GPP TS 29.244/23.501) 的映射。

### L1: 逻辑与物理视图
- **重点**：划分微服务边界 (Microservices) 与容器部署。
- **关联需求**：IR。
- **逻辑视图**：展示各服务间的 REST/gRPC 调用关系。
- **物理视图**：展示 Pod 的亲和性、资源限制 (Resource Limits) 以及 K8s 节点分布。

### L2: 开发与进程视图
- **重点**：深入单一服务内部，描述 C 模块与 Go 逻辑的交互。
- **关联需求**：SR/AR。
- **开发视图**：展示 VPP 插件架构、共享内存池 (Memory Pool) 设计。
- **进程视图**：展示 DPDK 线程模型 (Master/Worker Threading) 与无锁队列通讯。

详情参考 `docs/arch/views/` 下的模板。
