# 技术评审规范 (tech-review)

> **专家级工作流**: 评审准备度检查；TR 里程碑 Checklist 审计；评审结论闭环 (Go/No-Go)。

## 核心任务与操作指南
1. **启动评审**: 根据 `GEMINI.md` 阶段，确定当前的 TR 节点（TR1~TR6）。
2. **执行审计**:
   - 参照 [里程碑指南](references/guide.md) 中的 Checklist 逐项核对交付件。
   - 识别风险项并记录在 **IRTM** (缺陷追踪矩阵) 中。
3. **闭环决策**: 
   - **Go**: 满足 100% 关键项（P0），可以进入下一研发阶段。
   - **Conditional Go**: 存在非致命风险，但在规定时间内必须闭环。
   - **No-Go**: 核心规格不符，必须重回开发/设计环节。

## 闭环准则
- 所有的 TR 遗留问题必须关联需求 ID 并记入 **spec-qclm.md**。
- **Conditional Go 约束**: 允许进入下一阶段，但上一阶段遗留的所有 P0/P1 项必须在 5 个工作日内完成闭环，否则自动拦截后续质量门控。
- **No-Go 处理**: 必须执行“重修”流程：
    1. 提交 RCR (变更单) 记录修正逻辑。
    2. 完成修复后，必须重新执行本阶段的全部 Checklist 审计。
- 只有 TR5 通过后，方可启动正式发布流程。


## 交付契约 (Delivery Contract)
- **交付件 (Deliverables)**:
- docs/04-management/spec-decision-log.md
- docs/05-quality/verification/qa-audit-log.md
- **质量门限 (Quality Gate)**:
- 100% decision integrity
- Audit score > 95
- 0 P0 risks open
- **挂载里程碑 (Milestone)**: TR1
- **评审角色 (Reviewer)**: QA
