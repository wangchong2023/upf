# 服务与模块拆分原则 (svc-decomp) 规范指南 (v0.2)

## 1. L1 级微服务划分准则
划分微服务时必须基于以下维度评估：
- **需求内聚性**: 将关联紧密的 IR（特性）划归同一微服务。
- **弹性需求差异**: 吞吐量敏感型 (PERF) 与连接数敏感型 (FUN) 特性应考虑拆分。
- **故障隔离域**: 核心转发逻辑与非核心辅助逻辑应拆分。
- **技术栈匹配**: 矢量化报文处理 (VPP) 与通用业务逻辑 (Go) 应物理隔离。

## 2. 需求到架构的映射 (RTM 扩展)
微服务拆分决策必须建立与 IR 的映射矩阵：
| 微服务名称 (L1) | 承载的 IR 列表 | 划分理由 |
| :--- | :--- | :--- |
| example-svc | IR.UPF.FUN.001, IR.UPF.PERF.002 | 理由描述 |

## 3. 架构冲突与评审
在 L1 划分阶段，必须执行**“内聚/耦合”**评估，确保服务间没有循环依赖，且 API 版本管理策略符合 `api-doc` 规范。
