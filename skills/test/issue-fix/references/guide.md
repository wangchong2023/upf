# 缺陷管理与定位规范指南

## 1. 问题复现
- **环境隔离**: 在隔离的测试环境中通过 TRex 或模拟器复现故障流量。
- **数据收集**: 抓取核心转储 (Core Dump)、GDB 调试日志及 N4 协议包。

## 2. 根因分析 (RCA) 与溯源
每个 Bug 必须关联到：
- **受影响的 AR**: 哪个模块的逻辑导致了问题？
- **缺失的 TC**: 为何现有的 UT/IT 没能拦截到此 Bug？

## 3. 闭环管理
- **回归测试**: 修复代码前必须先编写一个失败的测试用例 (TC)，修复后该 TC 必须通过。
- **防止回退**: 每次修复的代码变更必须重新运行所有关联的 IT 场景，确保没有产生性能退化或功能回归。
