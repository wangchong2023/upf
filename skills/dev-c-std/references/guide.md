# C 模块设计规范指南 (整合华为规范 v3.1)

## 1. 头文件与接口设计 (华为规范精华)
- **自包含**: 每一个头文件必须能够独立编译，不依赖特定的包含顺序。
- **禁止循环依赖**: 禁止头文件 A 包含 B，同时 B 包含 A 的情况。
- **保护宏**: 必须使用 `#define FILENAME_H` 防止重复包含。
- **内联函数**: 只有及其简单的函数（< 5行）才允许在头文件中定义。

## 2. 代码风格与排版
- **大括号**: **[硬性]** 函数体、`if`、`while`、`for`、`switch` 的左大括号 `{` 必须另起一行。
- **括号完备性**: 逻辑表达式必须使用完备的括号（如 `if ((a == b) || (c == d))`），消除优先级歧义。
- **恒等比较**: 建议使用 `NULL == ptr`（左常量）风格防止误写为赋值。
- **缩进**: 统一使用 4 个空格，严禁使用 Tab 键。

## 3. 命名规范
- **变量与函数**: 统一采用 **unix-like snake_case (下划线命名法)**。示例：`session_context`, `get_packet_buffer()`。
- **宏与常量**: 必须全部大写，单词间用下划线分隔。示例：`MAX_PACKET_SIZE`。
- **全局变量**: 采用 snake_case 并增加 `g_` 前缀。示例：`g_active_session_count`。
- **结构体/类型**: 采用下划线命名，通常以 `_t` 结尾（如 `session_info_t`）或按模块约定命名。成员必须使用 snake_case。

## 4. 内存管理规则 (VPP 专用)
- **缓冲区释放**: 必须确保 `vlib_buffer_t` 在所有分支路径（包括错误分支）均被释放或转交。
- **内存置零**: 分配敏感数据内存后，必须立即使用 `memset` 或 `memset_s` 清零。
