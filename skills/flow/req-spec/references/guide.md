# 原始需求 (RR) 规范指南 (IPD 标准 v0.3)

## 1. 命名规范
- **格式**: `RR.项目名.序列号` (全部大写)
- **示例**: `RR.UPF.001`

## 2. 需求提取与 RR 转化
原始诉求通常来自 3GPP 标准、客户访谈、市场报告。
### 转化步骤:
1. **识别核心角色**: 谁发起的诉求？(如 UE, SMF, DN)。
2. **定义业务场景**: 在什么情况下发生的？(如 会话建立过程、异常掉线过程)。
3. **量化业务目标**: 最终要达到什么结果？(如 转发成功、计费准确)。

### RR 描述模版 (背景-现状-目标):
- **背景**: [基于什么协议或场景]
- **现状**: [当前系统的局限性或缺失的功能点]
- **目标**: [期望实现的功能或性能指标]
- **优先级**: [P0/P1/P2]

## 3. 职责
- 定义业务诉求的起点。
- 作为后续所有需求拆解的根溯源点。
