# 5G UPF 云原生 (K8s) 部署规范指南 (专家版 v1.0)

## 1. 高性能基础设施要求
- **CNI 选型**: 必须使用 **Multus CNI** 以支持多平面（管理平面、信令平面、数据平面）隔离。
- **SR-IOV**: 数据面 Pod 必须通过 SR-IOV Device Plugin 直接访问硬件 VF，禁止使用普通内核转发。
- **大页内存**: 必须在容器规格中显式申明 `hugepages-1Gi` 或 `hugepages-2Mi` 资源。

## 2. 调度与亲和性原则 (DfP/DfR)
- **CPU Pinning**: Pod 的资源申请必须设置为 `limits == requests`，以进入 K8s 的 **Guaranteed QoS** 类，确保 CPU 独占。
- **反亲和性**: 同一个会话组的 CP 与 DP 备份节点、以及 DDB 集群节点，必须配置 **podAntiAffinity**，严禁部署在同一物理机/可用区。

## 3. 容器安全与特权管理 (DfSec)
- **最小化权限**: 数据面 Pod 尽量避免使用 `privileged: true`。应通过 `capabilities: add: ["NET_ADMIN", "IPC_LOCK", "SYS_ADMIN"]` 精准授权。
- **镜像安全**: 必须通过 **mgr-ipd-sbom-audit** 确认镜像内不包含高危 CVE 漏洞库。

## 4. 业务连续性保证 (ISSV)
- **就绪探测 (Readiness)**: 控制面必须在完成 PFCP Node 关联及 DDB 同步后，方可将 Readiness 标记为 True。
- **平滑下线 (PreStop)**: Pod 退出前必须执行 `PreStop Hook`，通知 SMF 转移会话或通知对端节点进入 Graceful Restart。
