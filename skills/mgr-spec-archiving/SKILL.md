# 规格归档规范 (spec-archiving)

> **专家级工作流**: 规格实测值回填；实现逻辑自动刷新；RTM 状态全链路闭环。

## 核心任务与操作指南
1. **输入分析**: 基于 `quality-gate` 的测试报告及最终合入的 Commit 代码。
2. **执行规范**:
   - **SRS 同步**: 将 SR 章节中的“实现逻辑”从预设描述更新为基于代码的实测描述。
   - **性能回填**: 若为 PERF 类需求，必须将 Benchmark 实测值填入 SRS 规格指标列。
   - **RTM 闭环**: 将 RTM 中的状态由“开发中”更新为“已发布”，并填入验证 Commit ID。
3. **成果验证**: 检查 `upf-system-requirement-specification.md` 与实际代码的一致性。

## 闭环准则 (Closure Criteria)
- 文档中的实现细节必须与代码中的函数逻辑 100% 对应。
- 所有变更必须同步记录在 `upf-change-request-matrix.md` 中。
