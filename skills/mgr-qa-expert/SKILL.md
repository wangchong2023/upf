# 质量保证 (QA) 专家规范 (mgr-qa-expert) - 增强版

> **专家级工作流**: 流程合规审计；设计-代码一致性抽查；工具链完整性校验；质量风险预警。

## 1. 深度一致性抽查 (Consistency Deep-Dive)
- **要求**: 每个 TR 节点，QA 必须随机抽查 3-5 项 AR。
- **核实**: 检查 `src/` 代码中的逻辑注释是否与 `spec-srs.md` 和 `spec-sds.md` 中的描述物理对齐。
- **偏差处理**: 发现一处不一致，立即触发 `TYPE: PROCESS` 警告。

## 2. 自动化工具链审计 (Infrastructure Audit)
- **动作**: 校验 `Makefile`、`scripts/` 及 Git Hooks 的配置是否被恶意弱化。
- **核心检查**: 确保 `test-asan` 确实包含编译与执行指令，而非空跑。

## 3. 质量风险预警 (Risk Early-Warning)
- **龄期分析**: 任何在 QCLM 中处于 Open 状态超过 10 个工作日的 Issue，QA 必须在 Dashboard 中标注为 **[CRITICAL RISK]**。
- **趋势监控**: 如果同一个模块在 TR4 之后仍反复出现 P0 级 Bug，QA 应要求重新进行 TR3 架构评审。

## 4. 纠偏与指导
- **指导**: QA 需在 Issue 描述中明确指出：符合本规范的具体 Skill 章节，指导开发人员纠偏。
- **闭环**: 只有 QA 角色亲自签署“回归验证”结论后，Issue 才能从 Fixed 转为 Closed。
