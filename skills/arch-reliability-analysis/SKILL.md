# 可靠性分析规范 (reliability-analysis)

> **专家级工作流**: DFMEA 失效模式识别；FTA 故障树建模；RPN 风险排序；设计改进闭环。

## 核心任务与操作指南
1. **DFMEA 实施**:
   - 针对当前架构组件，识别“失效模式 -> 失效后果 -> 失效原因”。
   - 计算 **RPN = 严重度(S) × 发生频度(O) × 探测度(D)**。
   - 对 RPN > 100 的项，必须在 `SRS` 或 `ADR` 中增加“设计控制建议”。
2. **FTA 实施**:
   - 定义“顶层不希望事件”（如：UPF 进程 Panic）。
   - 使用与门/或门向下推演基本事件。
   - 识别单点故障（Single Point of Failure），反馈至 `ha-dr` 方案。
3. **成果应用**: 将分析出的缓解措施转化为具体的 **AR (分配需求)**。
