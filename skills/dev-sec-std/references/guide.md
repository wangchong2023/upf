# 安全编码规范指南 (整合华为安全规范 v3.1)

## 1. 变量与初始化 (防信息泄露)
- **赋初值**: 所有的指针、描述符及 BOOL 变量在声明时必须赋予初值（如 `NULL` 或 `INVALID`）。
- **清零原则**: 存储过口令、密钥等敏感信息的内存，在释放前必须强制清零。

## 2. 字符串与数组操作 (防缓冲区溢出)
- **长度校验**: 数组下标操作前必须进行边界检查；禁止将不受信的字符串复制到固定长度数组。
- **安全函数**: 严禁使用 `strcpy`, `strcat`, `sprintf`；必须使用 `strncpy_s`, `strncat_s`, `snprintf_s`（或对应安全版本）。
- **截断处理**: 字符串操作后必须手动确保以 `\0` 结尾。

## 3. 外部数据控制 (防注入攻击)
- **合法性校验**: 所有来自 N4 接口或配置接口的数据，必须检查范围、长度和格式。
- **循环控制**: 禁止使用不受信的外部数据直接作为 `for/while` 循环的次数上限。
- **格式化安全**: 禁止将外部输入直接作为 `printf` 系列函数的第一个参数（格式化字符串）。

## 4. 整数安全
- **算术溢出**: 对外部输入的整数进行加减乘运算前，必须预判是否会发生溢出。
- **位运算**: 禁止对有符号整数进行位操作。
