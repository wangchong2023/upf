# 5G UPF 技术评审 (TR) 里程碑 Checklist (专家版 v1.0)

## TR1: 概念与需求评审
- [ ] **协议对齐**: 所有 RR/IR 必须明确关联 3GPP R15/R16/R17 具体的 TS 编号。
- [ ] **基线达成**: 100% 的特性已录入 SRS 并生成 RTM 初始链条。
- [ ] **冲突分析**: 确认不同运营商需求间是否存在逻辑冲突。

## TR2: 计划与方案评审
- [ ] **架构 ADR**: 核心选型（VPP vs DPDK, SDB 选型）必须有 Alternatives 比较记录。
- [ ] **可靠性基石**: DFMEA 报告必须输出，RPN > 100 的风险项必须有对应的 AR 缓解措施。
- [ ] **硬件规划**: CPU 核心分配、大页内存及 SR-IOV 网卡规格已在总体方案中明确。

## TR3: 概要设计评审
- [ ] **接口契约**: 微服务间接口定义 (Protobuf/OpenAPI) 必须在仓库中锁定，并生成 Mock。
- [ ] **数据建模**: 关键业务对象（Session, Context）的 DDB 存储结构已完成 Schema 评审。
- [ ] **可服务性 (DfS)**: 错误码字典及日志分级标准已在设计文档中明确。

## TR4: 详细设计与单元验证评审
- [ ] **白盒准入**: `make lint-c` 和 `make lint-go` 必须 0 报错（包含 cppcheck/clang-tidy）。
- [ ] **TDD 覆盖**: 100% 的 P0 级 AR 必须关联已执行成功的 UT 用例；覆盖率 > 80%。
- [ ] **安全编码**: 华为安全规范中的“敏感内存清零”及“输入校验”逻辑已在代码评审中通过核实。

## TR5: 样机与集成测试评审
- [ ] **性能基准**: 性能 Benchmark 报告已输出，实测值必须达到 SRS 设定目标的 95% 以上。
- [ ] **故障自愈**: 现场验证了 Pod 重启后的会话恢复时延（要求 < 3s）。
- [ ] **缺陷闭环**: IRTM 中 100% 的 P0/P1 缺陷必须处于 Closed 状态；剩余 P2 缺陷比例 < 5%。

## TR6: 正式发布评审
- [ ] **规格归档**: `spec-archiving` 完成，SRS 已更新为包含实测值的 Release 版本。
- [ ] **运维闭环**: SOP 维护手册、故障排查字典、黑匣子解析工具已同步发布。
- [ ] **开源合规**: SBOM 清单已生成，100% 外部依赖库已完成 CVE 漏洞扫描。
