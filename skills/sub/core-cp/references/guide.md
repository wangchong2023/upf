# 核心网控制面 (UPF-C) 研发指南

## 1. 分配需求 (AR) 落地
- **AR 识别**: 在编写代码前，必须明确当前模块承载的 AR 编号（如 AR-01-01-C）。
- **逻辑实现**: 代码实现必须完整覆盖 AR 中定义的功能点。
- **边界条件**: AR 中定义的输入（如 PFCP 消息 IE）和输出（如会话管理对象）必须有严格的校验逻辑。

## 2. 核心开发规范
- **协议栈遵循**: 严格对标 3GPP TS 29.244 标准。
- **状态机设计**: 每一个 PDU 会话必须有清晰的状态管理逻辑。
- **错误处理**: 对异常的 PFCP 消息必须有明确的错误响应码。
