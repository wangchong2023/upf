# 核心网数据面 (UPF-D) 研发指南

## 1. 分配需求 (AR) 落地
- **AR 识别**: 所有的转发面逻辑必须有明确的 AR 编号对应（如 AR-02-01-D）。
- **性能约束**: 数据面 AR 通常带有严格的性能指标要求（吞吐量、时延）。
- **规则下发对接**: 数据面 AR 必须考虑如何接收并应用来自控制面 (UPF-C) 的规则。

## 2. 转发流水线逻辑
- **GTP 处理**: 实现高效的封装与解封装逻辑。
- **PDR/FAR 匹配**: 采用高速查找算法。
- **QER 限速**: 实现基于令牌桶等算法的速率控制。
- **统计统计**: 实现基于 URR 的原子计数器统计。
