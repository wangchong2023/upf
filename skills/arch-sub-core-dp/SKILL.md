# 数据面子系统设计规范 (arch-sub-core-dp) - 增强版

## 1. VPP 原子 Node 处理准则
所有的转发插件 Node 必须遵循以下“原子化”要求：
- **[DfP] 双循环优化 (Dual-Loop)**: 关键 Node 必须实现四报文并发处理（SIMD 友好）。
- **[DfT] 打点累加器**: 每个 Node 必须内置 `error_counters`，记录 Drop、Success 及特定协议错误数。
- **[DfSec] 预取与边界**: 执行 `vlib_buffer_advance` 前必须执行 `vlib_buffer_is_valid` 校验。

## 2. 内存模型
- 必须优先使用 `vlib_mempool`。
- 多核共享数据必须通过 `clib_atomic` 系列宏进行原子操作。
