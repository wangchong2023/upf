# 控制面子系统设计规范 (arch-sub-core-cp) - 增强版

## 1. 分层状态机设计 (Tiered FSM)
- **要求**: 必须将底层协议状态（PFCP Heartbeat/Path）与业务状态（PDU Session）物理分离。
- **机制**: 业务状态机应基于事件驱动，严禁在状态处理函数中执行阻塞式 IO。

## 2. 安全 IE 遍历模式
- **规范**: 解析 PFCP 变长 IE 时，必须使用“安全步进器”模式，禁止直接使用指针加法。
- **校验**: 递归解析 Grouped IE 时，必须设置最大深度限制，防止恶意报文导致的栈溢出。
