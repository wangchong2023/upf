# 缺陷修复规范 (test-issue-fix) - 工业级版

> **专家级工作流**: 根因分析报告；方案比选 (Alternatives)；架构师审批 (Approval)；TDD 修复；规范化提交。

## 核心任务与操作指南
1. **定位与记录**: 在 `spec-qclm.md` 记录 Issue，使用 `systematic-debugging` 锁定根因。
2. **方案比选 (关键)**: 
   - 必须提出至少两套修复方案。
   - 评估对 DfP (性能) 和 DfR (可靠性) 的影响。
   - **必须获取人工 Approve 后方可实施。**
3. **修复实施 (TDD)**:
   - **RED**: 编写能复现 Bug 的失败回归用例。
   - **GREEN**: 实现已批准的方案。
4. **验证与门控**: 运行 `make quality-gate` 确保 ASan 扫描通过且文档一致。
5. **规范提交**: Commit Message 必须遵循 `fix(模块): [IssueID] 简述`。

## 闭环准则
- 只有在 `spec-qclm.md` 中回填了修复 Commit 且 QA 确认后，Issue 才能 Closed。
