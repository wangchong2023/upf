# 可服务性 (DfS) 详细设计指南

## 1. 统一日志与故障码
- **要求**: 所有的 ERROR 级别日志必须遵循 `[MODULE-ID][ERROR-CODE] Description` 格式。
- **目标**: 实现故障的快速自动检索与自动化 SOP 匹配。

## 2. 黑匣子 (Blackbox) 与死后分析
- **要求**: 数据面发生 Panic 或异常重启前，必须将当前的 `vlib_buffer` 上下文和寄存器快照写入持久化存储。
- **验证**: 模拟崩溃，检查 `/var/log/upf/crash/` 下是否有解析可用的堆栈文件。

## 3. 在线升级 (ISSV) 设计
- **要求**: 支持 K8s 滚动更新，且双机热备组网下必须实现“会话无感知切换”。
- **目标**: 升级期间 PDU Session 掉话率 < 0.01%。

## 4. 资源自监控
- **要求**: 必须实时上报 CPU 利用率、内存水位、大页内存剩余量。
- **阈值**: 达到 85% 触发告警，达到 95% 触发流量过载控制。
