# 可服务性设计规范 (dfs-std)

> **专家级工作流**: 日志分级标准；自诊断接口设计；无损升级方案；黑匣子 (Post-mortem) 机制。

## 核心任务与操作指南
1. **输入分析**: 在 `adr-decision` 或 `svc-decomp` 阶段同步介入。
2. **执行规范**:
   - **日志规范**: 必须定义 Trace/Debug/Info/Warn/Error 五级，且 Error 日志必须携带故障定位码。
   - **自诊断**: 每个微服务必须提供 `/health` (心跳) 和 `/dump` (运行时状态) 接口。
   - **可观测性**: 关键转发路径必须埋点，支持 Prometheus 指标导出。
3. **成果验证**: 通过 `sop-manual` 模拟故障处理，验证 DfS 设计的有效性。
