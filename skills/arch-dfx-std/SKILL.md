# 卓越设计规范 (arch-dfx-std)

> **专家级工作流**: 预防性设计；DfS (易维)；DfR (稳健)；DfT (易测)；DfP (高性能)。

## 1. DfS: 可服务性 (Serviceability)
- **自诊断**: 必须提供 RESTful /health 和 /stats 接口。
- **故障码**: ERROR 级别日志必须包含标准故障码。
- **在线升级**: 支持无损滚动更新 (ISSV)。

## 2. DfR: 可靠性 (Reliability)
- **过载保护**: 针对 N4 控制流和 N3 数据流必须实现限流 (Rate Limiting)。
- **反压机制**: 缓冲区水位达到 85% 时必须触发上游反压信号。

## 3. DfT: 可测试性 (Testability)
- **Mock 预留**: 外部接口（如 DDB, NRF）必须支持环境变量驱动的 Mock 模式。
- **打点统计**: 关键业务路径必须内置累加器，通过 CLI 查看实时处理数。

## 4. DfP: 性能设计 (Performance)
- **零拷贝**: 严禁在转发平面进行不必要的报文内存拷贝。
- **亲和性**: 必须定义逻辑核心与硬件网卡队列的绑定矩阵。

## 5. DfSec: 安全架构 (Security)
- **平面隔离**: 控制面、数据面、管理面网络平面必须在逻辑和物理上严格隔离。
