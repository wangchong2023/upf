# 开源治理规范 (mgr-oss-governance)

> **专家级工作流**: 开源准入申请；License 黑白名单核实；社区成熟度评估；SBOM 动态审计。

## 1. 开源准入原则 (事前控制)
- **选型标准**: 必须优先选择 Apache 2.0, MIT, BSD 协议。
- **成熟度要求**: GitHub Star > 1k 且近 3 个月有持续 Commits。
- **替代性分析**: 核心转发逻辑应尽量自研或基于 VPP 官方插件，减少对不可控第三方库的依赖。

## 2. 许可证红线 (License Red-line)
- **严禁使用**: GPLv3, AGPL (传染性协议，可能导致私有代码被迫开源)。
- **限制使用**: LGPL (仅限作为动态链接库使用)。

## 3. 全生命周期管理
- **TR2 准入**: 提交《开源组件选型报告》，包含 CVE 历史记录。
- **TR6 审计**: 通过 `make sbom` 验证物理实现与准入清单的一致性。

## 4. 纠偏动作
- 发现违规协议，由 QA 触发 **IS.QA.OSS** 缺陷，强制要求在发布前替换或进行隔离封装。
