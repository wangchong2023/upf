# 代码评审规范指南

## 1. C/VPP 评审 Checklist
- **内存泄漏**: 所有分配的 `vlib_buffer_t` 或 `mempool` 对象是否正确释放？
- **并发冲突**: 共享数据访问是否使用了原子变量或无锁队列？
- **性能**: 是否存在不必要的系统调用或大规模内存拷贝？
- **安全**: 是否对数据面报文的头部偏移量进行了越界检查？

## 2. Go 评审 Checklist
- **并发管理**: goroutine 的生命周期是否受 Context 控制？是否存在协程泄漏？
- **错误处理**: 所有的 `error` 是否被正确检查？是否包含足够的上下文信息？
- **接口设计**: 是否使用了最小接口设计原则？

## 3. 合入条件
- **Linter**: 所有的静态检查必须全部通过。
- **Coverage**: 测试覆盖率必须不低于 AR 定义的基准线。
- **Approve**: 至少由一名架构师或核心研发评审通过。
